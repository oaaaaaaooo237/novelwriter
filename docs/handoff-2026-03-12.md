## 用户已明确提出、但还没做的高优先需求

### 1. 内置 Markdown 编辑器

用户明确要求：

- 不要主要靠系统外部程序打开 `.md`
- 要在 GUI 里直接编辑项目文档

下一步建议：

- 在 GUI 中加入内置 Markdown 编辑区
- 支持打开、编辑、保存
- 快捷入口中的 `.md` 默认改为内置编辑器打开

### 2. 自动生成总章与各章摘要，供用户确认

用户明确要求：

- 输入书名、题材、premise 后
- 软件自动生成总章摘要和各章一句摘要
- 由用户确认，而不是从空白开始手填

下一步建议：

- 在状态层增加自动章节摘要骨架生成
- 在 GUI 中增加“自动生成并确认”页面
- 把这一项视为高优先，不要再让用户从零填摘要

## 建议的下一步开发顺序

如果继续开发，建议不要先切版本，先把 `codex/v0.5.0` 做完整，再发布：

1. 先完成 GUI 可视化收尾
   - 内置 Markdown 编辑器
   - 当前章节面板 / 审校面板优化

2. 完成自动摘要确认流
   - 输入 premise 后自动生成
   - GUI 里确认与修改

3. 再考虑是否发布 `v0.5.0`

如果这两项还没做，就不要急着发布 `v0.5.0`。
