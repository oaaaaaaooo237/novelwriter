# novel-writer

> 专业长篇小说 AI 写作助手 | Professional Novel Writing Assistant

[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)
[![Version](https://img.shields.io/badge/Version-1.0-blue.svg)](./SKILL.md)

一个专为 **50万-200万字** 长篇连载小说设计的 AI 写作系统。具备三层记忆系统、伏笔追踪、剧情起伏检测等高级功能，确保故事线完整、剧情精彩、绝不遗忘。

---

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🧠 **三层记忆系统** | 记住所有历史内容，永不遗忘 |
| 📖 **完整故事线** | 大纲管理，章节规划 |
| 🎭 **伏笔追踪** | 记录所有埋下的坑，确保后续填坑 |
| 📈 **剧情起伏检测** | 监控爽点/高潮/平缓节奏 |
| ✂️ **详略控制** | 该详写详写，该略写略写 |
| 👥 **人物状态追踪** | 记录每个角色的当前位置和状态 |

## 🚀 快速开始

### 第一步：安装 Skill

将本仓库克隆到你的 OpenCode skills 目录：

```bash
# 克隆仓库
git clone https://github.com/oaaaaaaooo237/novelwriter.git

# 移动到 OpenCode skills 目录
cp -r novelwriter ~/.config/opencode/skills/novel-writer
```

### 第二步：告诉 AI 你的小说保存位置

> **"请告诉我你的小说项目保存文件夹路径，例如：D:/MyNovels/故事名 或 C:/Users/你的名字/Documents/Novels"**

### 第三步：选择模型

AI 会询问你想使用的模型：

> **你想用哪个模型来写？我推荐 MiniMax M2.1，性价比最高。**

可选模型：
- MiniMax M2.1（性价比最高 ¥8.4/百万 tokens）
- GLM-4（¥16/百万 tokens）
- Gemini 2.0 Flash（$0.6/百万 tokens）
- Claude 3.5（$15/百万 tokens）

### 第四步：开始写作

告诉 AI：
- "帮我写第1章"
- "我想写一个玄幻小说"
- "开始写小说"

## 🎭 伏笔追踪系统

### 伏笔类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **人物伏笔** | 人物背景、秘密 | "他手上那道疤..." |
| **物品伏笔** | 关键物品出现 | "那枚戒指..." |
| **事件伏笔** | 未来事件铺垫 | "三个月后的约定..." |
| **对话伏笔** | 言语中的暗示 | "小心身边人..." |
| **环境伏笔** | 场景细节铺垫 | "墙上挂着..." |

### 伏笔记录格式

```markdown
# 伏笔追踪清单

## 待填伏笔

| ID | 类型 | 章节 | 内容 | 状态 | 计划填坑章节 |
|----|------|------|------|------|-------------|
| V001 | 人物 | 第3章 | 男主父亲的神秘过去 | 待填 | 第15章 |
| V002 | 物品 | 第5章 | 女主丢失的玉佩 | 待填 | 第8章 |

## 已填伏笔

| ID | 类型 | 章节 | 内容 | 填坑章节 |
|----|------|------|------|---------|
| V003 | 事件 | 第1章 | 门派比武 | 第10章 |
```

## 📈 剧情起伏模型

### 节奏周期

```
高潮 ← 发展 ← 铺垫 ← 低潮 ← 对抗 ← 铺垫 ← 高潮...
```

### 检测标准

| 类型 | 特征 | 字数占比 |
|------|------|---------|
| **铺垫** | 背景介绍、日常描写、情感积累 | 30-40% |
| **发展** | 矛盾产生、冲突升级 | 20-30% |
| **高潮** | 核心冲突爆发、重大转折 | 10-15% |
| **解决** | 冲突结束、新局面 | 10-15% |
| **过渡** | 章节间衔接、情绪缓冲 | 5-10% |

### 写作要点

- **不要平铺直叙**：每 3000 字必须有一个小高潮
- **每章结尾要留钩子**：吸引读者继续看
- **详略结合**：日常可以略写，关键情节必须详写
- **情绪波动**：主角不能一直顺，也不能一直不顺

## ⚠️ 重要规则

### 黄金法则

1. **永远先读取记忆系统** - 每次开始新章节前，必须读取大纲、人物百科、近期记忆、伏笔清单
2. **永远更新记忆系统** - 每次写完章节后，必须更新本章概要、近期记忆、伏笔追踪
3. **永远检查伏笔** - 写新章节时，必须检查是否有之前埋下的伏笔需要呼应

### 章节结构要求

每个章节必须包含：
- 本章要点
- 正文
- 本章结束时的状态
