# 小说写作助手 (Novel Writer)

## ⚠️ 首次使用 - 重要说明

### 模板文件 vs 项目文件的区别

| 文件类型 | 位置 | 说明 |
|---------|------|------|
| **模板文件** | `~/.config/opencode/skills/novel-writer/templates/` | 保持不变，作为参考 |
| **项目文件** | 用户指定的小说文件夹 | 根据创作内容生成的实际文件 |

### 写作流程

```
1. 询问创作意图 → 2. 确认保存位置 → 3. 生成项目文件 → 4. 开始写作
```

---

## 一、系统概述

这是一个**专业长篇小说写作系统**，专为 50万-200万字的长篇连载小说设计。

### 核心能力

| 能力 | 说明 |
|------|------|
| 🧠 **三层记忆系统** | 记住所有历史内容，永不遗忘 |
| 📖 **完整故事线** | 大纲管理，章节规划 |
| 🎭 **伏笔追踪** | 记录所有埋下的坑，确保后续填坑 |
| 📈 **剧情起伏检测** | 监控爽点/高潮/平缓节奏 |
| ✂️ **详略控制** | 该详写详写，该略写略写 |
| 👥 **人物状态追踪** | 记录每个角色的当前位置和状态 |

## 二、开始写作前的问卷调查

在开始正式写作之前，**必须**通过交互式问答了解用户的创作意图。以下是完整的问卷流程：

### 第一阶段：基础信息（必须全部询问）

1. **小说标题/暂定名**
   > "你的小说叫什么名字？或者有没有暂定的标题？"

2. **小说题材类型**
   > "你想写什么题材的小说？比如：都市、玄幻、武侠、仙侠、科幻、历史、悬疑、言情、轻小说？"

3. **目标字数**
   > "你计划写多少字？比如：50万、100万、150万、200万+？"

4. **小说风格**
   > "你希望是什么风格？比如：热血、轻松、虐心、黑暗、升级流、系统流、日常、治愈？"

5. **目标读者群体**
   > "你的小说是给谁看的？比如：青少年、成年男性、成年女性、全年龄？"

### 第二阶段：核心创意（必须全部询问）

6. **一句话概括**
   > "用一句话概括你的故事核心卖点（类似电梯演讲）。比如：废柴少年意外获得上古传承，卷入正邪纷争，最终成为一代宗师的逆袭之路。"

7. **故事类型**
   > "你的故事属于哪种类型？比如：升级流、系统流、凡人流、洪荒流、穿越流、重生流、无敌流、迪化流？"

8. **核心冲突**
   > "全文最大的矛盾/冲突是什么？比如：正邪对立、种族战争、王朝更替、复仇、探宝、升级比赛？"

9. **主线剧情**
   > "主角为什么会踏上这条路？他的核心目标是什么？"

10. **最高潮点**
    > "你最想写的精彩爆发点在哪里？比如：越级战斗、英雄救美、揭露阴谋、决战boss？"

### 第三阶段：结局设定（必须全部询问）

11. **结局类型**
    > "你希望是什么结局？比如：大团圆（HE）、悲剧（BE）、开放式、还是没想好？"

12. **主角最终目标**
    > "主角的最终追求是什么？比如：成为最强、守护家人、复仇成功、探索真相、或者只是享受生活？"

### 第四阶段：世界观的深度（根据题材调整询问）

**如果选择玄幻/仙侠/武侠：**
- "修炼体系是什么样的？比如：练气→筑基→金丹→元婴→化神？或者有其他独特的体系？"
- "有哪些主要势力/门派？"
- "货币/资源体系是什么？比如：灵石、金币？"
- "这个世界有什么特殊规则或限制？"

**如果选择都市：**
- "故事发生在哪个城市/地区？"
- "主角的身份/职业是什么？"
- "涉及哪些行业/领域？"
- "有没有特殊背景（如隐藏身份、系统、重生）？"

**如果选择科幻：**
- "时代背景是近未来、远未来还是末世？"
- "科技水平如何？有哪些黑科技？"
- "人类 vs 什么势力？外星人、AI、变异生物？"

**如果选择历史：**
- "发生在哪个历史时期？"
- "是真实历史还是架空历史？"
- "主角身份是什么？平民、官员、还是？"

### 第五阶段：角色设定（根据回答深入询问）

**主角相关：**
- "主角的性格是怎样的？比如：腹黑、阳光、老实、果断、犹豫？"
- "主角有什么显著特点或金手指？比如：穿越者、重生者、系统拥有者、特殊体质？"
- "主角的优点和缺点分别是什么？"

**配角/女主相关（如果用户提到）：**
- "有没有明确的女主/男主设定？"
- "有哪些重要的配角或反派？"
- "反派智商怎么设定？是降智反派还是高智商反派？"

### 第六阶段：爽点/高潮点（必须全部询问）

13. **期待的核心爽点**
    > "你希望有哪些核心爽点？比如：装逼打脸、越级战斗、收集后宫、揭露阴谋、温馨日常、热血战斗？"

14. **雷点/毒点（必须避免的）**
    > "有没有特别雷的设定要避免？比如：圣母主角、绿帽情节、强行降智、反派太弱？"

### 确认保存位置

15. **项目保存路径**
    > "请告诉我你的小说项目保存文件夹路径，例如：D:/MyNovels/故事名 或 C:/Users/你的名字/Documents/Novels"

## 五、必须遵守的规则

### ⚡ 黄金法则

1. **永远先读取记忆系统**
   - 每次开始新章节前，必须读取：
     - `outline.md` (完整大纲)
     - `characters.md` (人物百科)
     - `memory/recent_chapters.md` (近期记忆)
     - `foreshadowing/tracker.md` (伏笔清单)

2. **永远更新记忆系统**
   - 每次写完章节后，必须更新：
     - `memory/chapter_XXX.md` (本章概要)
     - `memory/recent_chapters.md` (更新近期记忆)
     - `foreshadowing/tracker.md` (如有新伏笔)

3. **永远检查伏笔**
   - 写新章节时，必须检查是否有之前埋下的伏笔需要呼应
   - 如果有，在本章结尾或适当位置填坑

### 📝 章节结构要求

每个章节必须包含：

```markdown
## 第X章：章节标题

### 本章要点
- 情节点1
- 情节点2
- 需要呼应的伏笔

### 正文
[章节内容...]

### 本章结束时的状态
- 角色A：在哪里，做什么
- 角色B：情绪如何
- 伏笔：埋下了XXX
```

## 六、剧情起伏模型

### 节奏周期

```
高潮 ← 发展 ← 铺垫 ← 低潮 ← 对抗 ← 铺垫 ← 高潮...
```

### 检测标准

| 类型 | 特征 | 字数占比 |
|------|------|---------|
| **铺垫** | 背景介绍、日常描写、情感积累 | 30-40% |
| **发展** | 矛盾产生、冲突升级 | 20-30% |
| **高潮** | 核心冲突爆发、重大转折 | 10-15% |
| **解决** | 冲突结束、新局面 | 10-15% |
| **过渡** | 章节间衔接、情绪缓冲 | 5-10% |

### 写作要点

- **不要平铺直叙**：每 3000 字必须有一个小高潮
- **每章结尾要留钩子**：吸引读者继续看
- **详略结合**：日常可以略写，关键情节必须详写
- **情绪波动**：主角不能一直顺，也不能一直不顺

## 七、记忆系统详解

### 三层记忆架构

```
┌────────────────────────────────────────────────────────────┐
│  Layer 1: 当前层 (Always in Context)                      │
│  - 当前章节草稿                                              │
│  - 上一章最后3段                                            │
│  - 本章正在写的内容                                          │
├────────────────────────────────────────────────────────────┤
│  Layer 2: 近期层 (最近10章)                                │
│  文件: memory/recent_chapters.md                          │
│  - 每章一句话概要                                            │
│  - 关键情节点                                               │
│  - 人物状态变化                                             │
├────────────────────────────────────────────────────────────┤
│  Layer 3: 长期层 (全小说)                                  │
│  文件:                                                      │
│  - outline.md (完整大纲)                                    │
│  - characters.md (人物百科全书)                             │
│  - foreshadowing/tracker.md (所有伏笔)                     │
│  - world_settings.md (世界观设定)                          │
└────────────────────────────────────────────────────────────┘
```

### 文件命名规范

```
your-novel-folder/
├── outline.md              # 小说大纲（根据问卷生成）
├── characters.md           # 人物百科（根据问卷生成）
├── world_settings.md       # 世界观设定（根据问卷生成）
├── plot_summary.md         # 剧情概要（一句话概括全本）
├── memory/
│   ├── recent_chapters.md # 近期记忆（最近10章）
│   ├── chapter_001.md     # 第1章概要
│   ├── chapter_002.md     # 第2章概要
│   └── ...
└── foreshadowing/
    ├── tracker.md         # 伏笔总清单
    └── seeds/             # 伏笔详情
```

## 八、伏笔追踪系统

### 伏笔类型

| 类型 | 说明 | 示例 |
|------|------|------|
| **人物伏笔** | 人物背景、秘密 | "他手上那道疤..." |
| **物品伏笔** | 关键物品出现 | "那枚戒指..." |
| **事件伏笔** | 未来事件铺垫 | "三个月后的约定..." |
| **对话伏笔** | 言语中的暗示 | "小心身边人..." |
| **环境伏笔** | 场景细节铺垫 | "墙上挂着..." |

### 伏笔记录格式

```markdown
# 伏笔追踪清单

## 待填伏笔

| ID | 类型 | 章节 | 内容 | 状态 | 计划填坑章节 |
|----|------|------|------|------|-------------|
| V001 | 人物 | 第3章 | 男主父亲的神秘过去 | 待填 | 第15章 |
| V002 | 物品 | 第5章 | 女主丢失的玉佩 | 待填 | 第8章 |

## 已填伏笔

| ID | 类型 | 章节 | 内容 | 填坑章节 |
|----|------|------|------|---------|
| V003 | 事件 | 第1章 | 门派比武 | 第10章 |
```

## 十、上下文限制处理方案

### 核心问题

100万字小说无法一次性加载到模型上下文中。

**解决方案**：严格的三层记忆系统 + 分块加载

### 各模型上下文限制（参考值）

| 模型 | 上下文长度 | 有效写作长度 |
|------|-----------|-------------|
| MiniMax M2.1/M2.5 | 100K tokens | 约3-5万字/次 |
| GLM-4 | 128K tokens | 约4-6万字/次 |
| Gemini 2.0 Flash | 1M tokens | 约30-50万字/次 |
| Claude 3.5 | 200K tokens | 约8-10万字/次 |
| GPT-4o | 128K tokens | 约4-6万字/次 |

### 每次写作时的加载策略

每次写新章节前的加载流程（必须严格遵守）：

**Step 1: 加载长期记忆（约5K tokens）**
- outline.md（完整大纲，摘要版）
- characters.md（人物百科，关键角色）
- foreshadowing/tracker.md（伏笔清单）
- world_settings.md（世界观设定）

**Step 2: 加载近期记忆（约10K tokens）**
- memory/recent_chapters.md（最近10章概要）
- memory/chapter_XXX.md（上一章详细概要）

**Step 3: 加载当前上下文（约30K tokens）**
- 当前章节草稿
- 上一章结尾最后3段

**写作完成后：**
- 将本章概要写入 memory/chapter_XXX.md
- 更新 recent_chapters.md
- 更新伏笔 tracker
- 更新人物状态

### 章节长度控制

| 章节类型 | 建议字数 | 上下文占用 |
|---------|---------|-----------|
| 过渡章 | 2000-3000字 | 小，可多章连写 |
| 发展章 | 3000-5000字 | 中 |
| 高潮章 | 5000-8000字 | 大，需单独写 |
| 核心高潮 | 8000-15000字 | 极大，需分段写 |

1. **接受用户的修改意见**

- **玄幻** - 西方奇幻、异世界转生
- **武侠** - 传统武侠、新武侠
- **仙侠** - 修真、洪荒、凡人流
- **科幻** - 硬科幻、软科幻、赛博朋克
- **历史** - 架空历史、历史穿越
- **悬疑** - 推理、惊悚、恐怖
- **轻小说** - 日系轻小说风格

---

*本 Skill 专为长篇小说创作设计，确保故事线完整、剧情精彩、绝不遗忘。*
